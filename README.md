# ВелоУчёт

Локальная база для парка электровелосипедов на Laravel и SQLite.

## Запуск

Самый простой способ: запустите `start-velochet.cmd` двойным кликом. Скрипт запустит Laravel-сервер скрыто и откроет сайт в браузере.

Для остановки сервера запустите `stop-velochet.cmd`.

Ручной запуск:

```powershell
php artisan migrate
php artisan serve
```

Откройте `http://127.0.0.1:8000`.

Первая учетная запись администратора:

```text
Email: admin@velochet.local
Пароль: admin123
```

После первого входа измените пароль в разделе `Учетные записи`.

Основная база хранится в `database/database.sqlite`. Фотографии хранятся в `public/photo`.

Для полной резервной копии при остановленном приложении скопируйте:

- `database/database.sqlite`
- папку `public/photo`

## Организация учета

- Города содержат точки выдачи, возврата и хранения.
- Инвентарный номер создается автоматически в формате `EKB-MAIN-26-0001`.
- Клиент хранится в отдельной карточке и может иметь несколько аренд.
- При выдаче и возврате создаются отдельные акты состояния с пробегом, зарядом, замечаниями и фотографиями.
- Если при возврате указаны новые дефекты, велосипед автоматически переводится в статус `В сервисе`.
- Администратор видит все города и точки, управляет настройками и учетными записями.
- Сотрудник точки видит только данные своей точки и не имеет доступа к настройкам.

## Google Sheets

На страницах велосипедов, аренд и сервиса есть кнопка `CSV для Google Sheets`. Файл можно импортировать в Google Sheets без дополнительных настроек.

Автоматическая двусторонняя синхронизация тоже возможна, но для нее нужны Google Cloud Project, включенный Google Sheets API, service account и доступ этого аккаунта к нужной таблице. JSON-ключ нельзя добавлять в репозиторий; его следует хранить локально вне `public`.

Рекомендуемая схема: SQLite остается основной базой, а Google Sheets автоматически получает копию данных. Это исключает конфликты при одновременном редактировании сайта и таблицы.
